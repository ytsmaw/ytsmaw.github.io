Engineering Projects: Intuitive Synthesiizer Interface

The Problem

Since the 1960s, artists have been using analog synthesizers to produce music. However, synthesizers have always had complex and intimidating interfaces, making them difficult to use. The team spent several weeks interviewing composers and musicians who work heavily with synthesizers to come up with our concept. We considered developing new synthesizer modules, motion-based interfaces, and methods to connect synthesizers to auxilary lightshows. In the end, we found that the most persistant problem for synth users was the inability to recall signal paths that they have made. Different modules and filters are patched together using audio cables to create a final sound - a process that can take hours or even days. Once the artist decides to move on from a patch, it can become impossible to return to the original sound.

Our Solution

Our solution was to create a new synthesizer interface that stays true to the aesthetic style and analog nature of the original synthesizers from the 1960s and '70s. Once an artist has reached a patch that they like using the synthesizer, they can press a button to save it in their patch bank to return to it later. The knob positions are saved using encoders attached to the knobs and the switch states and signal path are saved by passing a high-voltage pulse through the synthesizer. The synthesizer can also be controlled through an Android application, which displays a visual representation of the final waveform and communicates with the synthesizer through bluetooth. We have filed a provisional patent application for our invention, on which I am listed as an inventor.

My Role

My role in the project was to build the android application to communicate with the synthesizer. I made the application exclusively in Android Studio, using the Java language for functionality and XML for the design of the interface. I was not familiar with Java when I started the app, and learned the language along the way with help from Stackoverflow and a short lynda course on object-oriented programming. I coded everything in the app, except for the bluetooth hardware code and the transfer function equation in the filter class. The application lets users create and manipulate the frequency and amplitude of sawtooth, triangle, and square waves; mix two waves together through superposition; apply a low-pass filter and increase the resonance of the signal; save patches in a bank; and communicate with the synthesizer over bluetooth to hear the result in real-time. The application does this through an interface that allows the user to modify a graphical representation of the wave on a touchscreen.
